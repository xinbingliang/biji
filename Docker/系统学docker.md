### CMD and ENTRYPOINT 

* RUN：执行命令并创建新的Image Layer

* CMD：设置容器启动后默认执行的命令和参数

  * 如果docker run指定了其他命令，CMD命令会被忽略
  * 如果定义了多个CMD，只有最后一个被执行

* ENTRYPOINT：设置容器启动时运行的命令

  * 让容器以应用程序或服务的形式运行

  * 不会被忽略，一定会执行

  * 最佳实践：写一个shell脚本作为entrypoint

    ````
    COPY docker-entrypoint.sh  /usr/local/bin
    ENTRYPOINT ["docker-entrypoint.sh"]

    EXPOSE 27017
    CMD ["mongod"]
    ````

#### Shell格式

```yaml

      - 8080:80

  my-bridge:
    driver: bridge
```

### Docker Secret

* 用户名密码
* SSH Key
* TLS认证
* 任何其他人不想看到的数据

1. 存储在Swarm Manager节点的Raft database中
2. Secret 可以assign给一个service，这个service就能看到这个secret
3. 在container内部Secret看起来像文件，但实际上在内存中

#### 管理和使用

##### 文件创建方式

* `vim password` 创建密码文件

  ````
  admin123
  ````

* `docker secret create my-password password`创建secret

* `rm password`然后删除`password`文件

* `docker secret ls` 查看secret

##### 标准输入

* `echo "adminadmin" | docker secret create my-password2 -`

##### 使用

* `docker service create --name client --secret my-password  busybox sh -c "while true; do sleep 3600;done"`
* `docker exec -it 934 sh` 进入容器
* `cd /run/secrets/` 
* `docker service create --name db --secret my-password -e MYSQL_ROOT_PASSWORD_FILE=/run/secrets/my-password mysql`
